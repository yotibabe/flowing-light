04 · Motion System

Duration

  • 快速反馈：150~220ms
  • 常规过渡：280~500ms
  • 叙事入场：700~1200ms

Easing

  • 推荐：ease-out / power3.out
  • 禁止：弹跳娱乐型缓动

Property Policy

  • 白名单：transform / opacity
  • 黑名单：width/height/margin/padding

Display Motion

展示页允许首屏分层入场，但不允许眩晕式持续动画。

生产引用：index Hero、archive 卡片入场

Reading Motion

阅读页动效总量必须收敛，并支持 reduced-motion 降级。

生产引用：episodes 内容区

05 · Pattern Library

Global Header Pattern

由 layout.js 注入，使用 data-base-path 与 data-nav-active 驱动，避免多页面手写漂移。

生产引用：index.html、archive.html、about.html、episodes/vol*/index.html

Hero Panel Pattern

展示页保留戏剧性，阅读页降低视觉干扰。桌面可用玻璃面板，移动端优先轻量化。

生产引用：index.html Hero、about.html 首屏叙事段

Reading Flow Pattern

导语、分节、正文、引用、收束 CTA 的连续阅读流，正文行长限制在 max-w-[800px]。

生产引用：episodes/vol1~vol4/index.html

Footer Pattern

页脚是“薄尾封”，统一薄内边距，不抢主体。详情页可通过 data-footer=none 关闭。

生产引用：layout.js + about local/footer global 配置

08 · Quality Gate

Do

  • 使用 text-fluid 与 gap-fluid 体系
  • 复用 layout.js 统一头尾
  • 保证交互态与焦点态可见
  • 移动端触控目标保持 44×44
  • 组件必须有生产引用与状态说明

Don’t

  • 跨页面手写独立 Header 样式
  • 硬编码颜色值与断点字号
  • 在移动端使用横向溢出导航
  • 动效直接修改 width/height/margin
  • 只做 UI 展示不写状态/场景边界
